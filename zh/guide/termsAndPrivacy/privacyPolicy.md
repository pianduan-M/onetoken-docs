**更新日期：2026年5月18日**

本隐私政策适用于 onetoken（域名：onetoken.one）提供的网站、API 中转、模型路由、统一 API 调用、账户管理、计费结算及相关企业级技术服务（以下统称“本服务”）。

onetoken 的服务定位是 API 中转与模型调用管理服务。我们不以存储、分析、训练、复用客户的业务内容为目的提供服务。本服务仅面向企业、社会组织、科研机构及其他组织客户，不面向自然人个人、消费者或未成年人提供个人用途服务。

您注册、访问、购买、充值、调用 API 或以其他方式使用本服务，即表示您及您所代表的组织理解并同意本隐私政策。如您未获得组织授权，或拟以个人目的使用本服务，请立即停止使用。

## 一、核心承诺：不存储 API 调用内容

为避免误解，onetoken 明确说明如下：

1. 我们不会存储您通过 API 提交的 prompt、问题、指令、系统提示词、上下文、对话历史、代码片段、文件正文、业务数据或其他请求正文内容。
2. 我们不会存储模型返回给您的回答、代码、分析结果、生成内容、上下文结果或其他响应正文内容。
3. 我们不会使用您的 API 调用内容训练 onetoken 自有模型，也不会主动将您的 API 调用内容提供给任何第三方用于模型训练。
4. 我们不会将客户的 API 调用内容作为知识库、样本库、案例库、数据集或运营分析素材进行保存或复用。
5. 我们不会主动人工查看您的 API 调用内容。除非法律法规、监管机关、司法机关依法要求，或您明确要求我们协助排查特定问题且另行提供相关内容。

本政策所称“API 调用内容”，包括但不限于：prompt、messages、上下文、用户问题、系统指令、工具调用参数中的业务正文、代码、文件正文、模型输出正文及其他可反映您实际业务内容的数据。

## 二、API 调用内容的处理方式

在正常服务流程中，API 调用内容仅用于完成您发起的当次请求：

1. 接收您的 API 请求；
2. 按照您选择或系统路由规则，将请求转发至相应模型或服务提供方；
3. 接收模型或服务提供方返回结果；
4. 将结果返回给您。

上述过程中，API 调用内容可能在网络传输、内存处理、请求转发、响应返回等环节被临时处理，但 onetoken 不会将其持久化保存到业务数据库、日志系统、样本库或训练数据集中。

如因特定企业客户的私有化部署、专属通道、定制排障、内容审计、合规留痕或其他特殊需求，需要改变上述处理方式，应由双方另行签署书面协议或在平台中明确开启相应功能。

## 三、我们会保留的必要服务记录

为提供 API 中转、计费、额度统计、安全风控、故障排查和合规运营，onetoken 需要保留必要的服务记录。该等记录原则上不包含 API 调用内容。

我们可能保留以下信息：

1. **账户与组织信息**：账号标识、登录名、邮箱或手机号、组织名称、账号角色、权限配置、认证状态、API Key 标识及其状态。API Key 原文会采取加密、哈希或其他安全措施处理。
2. **订单与计费信息**：订单记录、充值记录、余额、套餐、额度、消耗金额、发票信息、支付状态、合同或结算所需信息。
3. **调用元数据日志**：请求时间、请求 ID、账号 ID、组织 ID、API Key 标识、接口路径、模型名称或模型类型、Token 用量、计费金额、响应状态、错误码、延迟、调用来源 IP、User-Agent、频率限制记录。
4. **安全与风控日志**：登录记录、异常登录、异常调用、密钥风险、频率异常、攻击拦截、滥用风险、权限变更、系统告警和安全事件处理记录。
5. **工单与服务沟通记录**：您通过平台工单、账户后台或其他官方渠道主动提交的问题、截图、文件、说明和处理记录。

上述调用元数据日志用于证明调用发生、计算费用、排查接口错误、防止盗刷和滥用、保障系统安全，不用于还原、分析、保存或复用您的业务内容。

## 四、日志不包含的内容

除您另行主动提交给工单或双方另有书面约定外，onetoken 的常规调用日志不记录以下内容：

1. prompt、用户问题、系统提示词、上下文或对话历史；
2. 模型输出正文、生成代码、生成文本或回答内容；
3. 您提交的源代码、配置文件、业务文档、客户资料或文件正文；
4. 图片、音频、视频、文档解析后的正文内容；
5. 可直接还原您业务意图、业务逻辑、客户数据或商业秘密的请求体内容。

如发生接口错误，我们通常记录错误码、错误类型、请求 ID、模型类型、时间、延迟、Token 用量等排障所需元数据，而不是记录请求正文或响应正文。

## 五、第三方模型和服务提供方

onetoken 作为 API 中转和模型路由服务，可能根据您的选择、平台配置、模型可用性、价格、稳定性或合规要求，将请求转发至相应模型、算力、云服务或技术服务提供方。

请您理解：

1. onetoken 不保存 API 调用内容，不等于第三方模型或服务提供方完全不处理相关内容。第三方为完成模型推理和响应生成，可能会在其服务流程中处理请求内容。
2. 不同模型供应商的数据处理规则、日志规则、内容安全规则、地域规则和训练规则可能不同。您应根据自身业务、数据类型、客户承诺和监管要求选择合适模型。
3. 面向中国境内业务、用户或数据处理场景时，您应确保所选模型、供应商、调用区域和使用方式符合中国法律法规、行业监管、数据安全、个人信息保护、算法治理、生成式人工智能和深度合成等相关要求。
4. 如您需要仅使用中国境内模型、指定供应商、关闭某些模型、专属网关、私有化部署、日志隔离、内容不落盘承诺或单独数据处理协议，应在使用前与 onetoken 另行书面确认。

## 六、您不应提交的内容

即使 onetoken 不存储 API 调用内容，您仍应审慎使用本服务，并避免提交以下内容：

1. 国家秘密、工作秘密、重要数据或依法限制处理、传输、出境的数据；
2. 未经授权的个人信息、敏感个人信息、客户原始资料、身份信息、账户密码、密钥、银行卡信息、医疗健康信息、未成年人信息；
3. 商业秘密、核心代码、未公开财务数据、未发布产品资料、保密合同、源代码仓库内容或其他高敏业务资料；
4. 您无权上传、处理、转发或提供给第三方模型服务的任何内容；
5. 可能违反法律法规、监管要求、合同义务、保密义务或第三方权益的任何内容。

您及您所代表的组织应对 API 调用内容的合法性、授权来源、数据脱敏、模型选择、输出使用和下游业务后果承担责任。

onetoken 仅在必要范围内使用账户信息、计费信息、调用元数据和安全日志，用于以下目的：

1. 创建、验证、管理和保护账户；
2. 完成 API 中转、模型路由、额度统计、计费结算、订单管理和发票处理；
3. 监测服务可用性、排查接口故障、定位异常请求和优化网关性能；
4. 防止盗刷、欺诈、滥用、恶意调用、攻击、越权访问和账号风险；
5. 履行法律法规、监管要求、司法或行政机关依法提出的要求；
6. 处理争议、审计账单、核对调用量、维护 onetoken、客户和第三方的合法权益。

## 八、信息共享与披露

onetoken 不出售您的个人信息、账户信息、调用记录或业务数据。

我们可能在以下必要情形中共享或披露有限信息：

1. **为完成服务所必需**：向模型供应商、云服务商、网络服务商、支付机构、短信或邮件服务商、风控服务商、工单系统等提供完成服务所需的有限信息。

3. **依法合规**：根据法律法规、监管机关、司法机关、行政机关或其他有权机关依法提出的要求披露。
4. **保护合法权益**：为防止欺诈、攻击、盗刷、违法违规、系统滥用或安全事件，在必要范围内向相关方披露风险信息。
5. **业务变更**：如发生合并、分立、收购、资产转让、重组或类似交易，必要信息可能依法转移，但受让方应继续受本政策或不低于本政策保护水平的安排约束。

## 九、数据安全措施

onetoken 会采取与 API 中转服务性质相匹配的技术和组织措施保护信息安全，包括但不限于：

1. 使用 TLS/SSL 等方式保护传输安全；
2. 对账号、API Key、权限、后台访问和敏感配置进行访问控制；
3. 避免在常规日志中记录请求正文和响应正文；
4. 对必要日志和配置采取加密、脱敏、权限隔离或最小化处理；
5. 监测异常登录、异常调用、攻击、滥用和系统风险；
6. 建立必要的备份、审计、应急响应和安全事件处理机制。

您应妥善保管账号、密码、API Key、访问令牌、回调地址、服务器环境和第三方集成配置。因您或您所代表组织管理不当导致的泄露、盗用、损失或争议，由您及您所代表组织自行承担责任。

## 十、数据保留与删除

1. API 调用内容不作为 onetoken 常规存储对象，正常情况下不会被 onetoken 持久化保存。
2. 账户信息、订单信息、计费记录、调用元数据日志、安全日志和工单记录，会在实现本政策所述目的所需的合理期限内保留，并受法律法规、会计审计、争议解决、安全风控和合同履行要求约束。
